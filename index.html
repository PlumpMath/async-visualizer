Channels

A channel is a lot like a pipe, but with a few unique properties. It takes objects from one side and passes through to the other. It has a few unique properties, though. A standard channel will halt the producer side if there is no receiver on the other end of the channel. Similarly, receivers will wait until they are provided with messages. This provides a kind of built-in backpressure.

Buffered Channels

It's possible to make a buffered channel, which is a channel that does not block the producing side. Some buffers will never block, but will lose messages when the buffer is full. There three kinds of buffers:

  • A blocking buffer that behaves like an unbuffered channel when it is full. (chan (buffer 3))
  • A sliding buffer, which will drop the oldest messages when the buffer is full. (chan (sliding-buffer 3))
  • A dropping buffer, which will drop any new messages when the buffer is full. (chan (dropping-buffer 3))
